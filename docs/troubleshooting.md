# 常见问题与排查

## 识别不到账号

先确认当前版本支持的三段式格式：

```text
GPT邮箱----GPT密码----OTP取码源
```

注意中间字段是 GPT/OpenAI 登录密码，不是邮箱密码。

## 大部分账号不需要验证码，少数跳验证码

这是正常行为。GPT2JSON 默认优先走账密登录；只有服务端进入验证码页面时，才会访问取码源。

## 验证码超时

可能原因：

- 取码 URL 返回旧验证码；
- 取码源延迟较高；
- 取码源需要额外参数；
- 服务端发送验证码较慢。

处置：

1. 客户端会先对单账号执行自动重试，并在仍未成功时触发自动重跑补救；
2. 批次结束后，如果仍有可恢复失败，客户端会生成 `failed_rerun.secret.txt`，并由批次级自动补跑默认读取 1 次，把这些账号作为新批次补跑；高级选项可关闭或调到最多 3 次；
3. 批次级自动补跑后若仍有可恢复失败，GUI 中会保留“重跑失败账号”；
4. 点击“重跑失败账号”后，客户端读取同一份 `failed_rerun.secret.txt`，只载入这批失败账号，预检查通过后作为新批次执行；
5. 如果仍失败，再增大高级选项里的验证码等待超时或自动重跑补救次数；
6. 检查 `_diagnostics/failure_report.safe.json` 的分类和建议；
7. 确认取码 URL 能返回当前账号的新验证码。

如果取码源一次返回多个验证码，客户端会优先按时间字段选择最新验证码；没有时间字段时会更偏向后出现的验证码，并继续过滤登录前预取到的旧码。

## Callback 换 JSON 超时

客户端会先自动重试可恢复的网络超时，并进入单账号自动重跑补救；批次结束后，批次级自动补跑会默认读取 1 次 `failed_rerun.secret.txt`，只把可恢复失败账号作为新批次补跑。若仍失败，可以：

- 点击 GUI 中的“重跑失败账号”，读取同一份 `failed_rerun.secret.txt`，只补跑本次可恢复失败清单；
- 增大 HTTP 请求超时；
- 增加自动重跑补救次数；
- 降低并发数。

## 导出校验显示“不建议导入”

导出完成后，客户端会检查 Sub2API / CPA JSON 的关键结构：

- Sub2API：`accounts`、`credentials.access_token`、`credentials.refresh_token`、`client_id`、`expires_at`、`model_mapping` 等；
- CPA：`type=codex`、`email`、`expired`、`access_token`、`refresh_token` 等。

如果日志提示“不建议导入”，先不要导入目标系统。打开本次结果目录中的 `_diagnostics/summary.json`，查看 `export_validation.errors`，然后重新导出或调整输入数据。

## `failed_rerun.secret.txt` 是什么

当某些账号经过自动重试、自动重跑补救以及批次级自动补跑后，失败原因仍属于可恢复失败（例如旧验证码、OTP 超时、Callback 超时、HTTP 429/5xx）时，客户端会在对应批次目录中生成或保留：

```text
failed_rerun.secret.txt
```

它同时服务于批次级自动补跑和 GUI 的“重跑失败账号”按钮：批次级自动补跑会在批次结束后默认自动读取 1 次；手动点击“重跑失败账号”会把这些账号作为新的粘贴输入并开始新批次，不需要用户从原始账号文件里手工筛选。

注意：这个文件包含原始账号行，可能包括 GPT 登录密码和完整取码源，因此是敏感文件；不要上传到 GitHub、工单、聊天截图或公开日志。

## 找不到最新导出文件

桌面版默认输出目录是程序所在目录下的 `output/`。例如安装版通常是：

```text
<安装目录>/GPT2JSON/output/
```

如果你在 GUI 中手动选择过其它输出目录，客户端会记住该目录；再次修改输出目录后也会继续记住最新选择。

从 v0.1.2 开始，每次导出都会新建唯一批次目录：

```text
output/GPT2JSON_20260429_043512_a1b2c3/
```

GUI 完成后点击“打开输出目录”会优先打开本次结果目录。CLI 可在 `_diagnostics/summary.json` 中查看 `out_dir`；普通导入只需要根目录的 `sub2api_accounts.secret.json` 和 / 或 `CPA_<批次>/`。

## CPA 文件是不是一个账号一个 JSON

是。`CPA_<批次>/` 目录中一个账号一个 JSON；目录名随本次批次唯一化，新版不再额外打包 ZIP，直接使用这个文件夹更方便。

## 软件是否写注册表

源码版和便携版不会把 GUI 偏好写入注册表。安装器版本会创建标准 Windows 卸载项和快捷方式，这是安装器的正常行为。

## 安装器能否识别升级

可以。安装器会读取 Windows 标准卸载项中的安装路径和版本号；如果检测到本机已有 GPT2JSON，会默认使用原安装目录，并显示升级 / 修复模式。
